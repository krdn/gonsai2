## 시작하기

### 시스템 요구사항

- **브라우저**: Chrome, Firefox, Safari, Edge (최신 버전)
- **네트워크**: 백엔드 서버 및 n8n 인스턴스 접속 가능
- **해상도**: 1280x720 이상 권장

### 접속 방법

1. 웹 브라우저에서 프론트엔드 URL 접속 (기본: `http://localhost:3000`)
2. 로그인 페이지로 자동 리다이렉트
3. 계정으로 로그인 또는 회원가입

### 주요 화면 구성

```
┌─────────────────────────────────────────────────┐
│  Header (로고, 내비게이션)                      │
├─────────────┬───────────────────────────────────┤
│             │                                   │
│  Sidebar    │  Main Content                     │
│  (메뉴)     │  (페이지별 콘텐츠)                │
│             │                                   │
│             │                                   │
└─────────────┴───────────────────────────────────┘
```

## 로그인 및 회원가입

### 로그인

**경로**: `/login`

**절차**:

1. 이메일 주소 입력
2. 비밀번호 입력
3. "로그인" 버튼 클릭
4. 성공 시 `/workflows` 페이지로 이동

**기능**:

- ✅ 자동 로그인 (Remember Me)
- 🔐 세션 기반 인증
- ⚠️ 오류 메시지 표시 (잘못된 인증 정보)

### 회원가입

**경로**: `/signup`

**절차**:

1. 이메일 주소 입력
2. 비밀번호 입력 및 확인
3. (선택) 추가 프로필 정보 입력
4. "회원가입" 버튼 클릭
5. 성공 시 자동 로그인

**요구사항**:

- 이메일: 유효한 이메일 형식
- 비밀번호: 최소 8자 이상 (영문, 숫자, 특수문자 조합 권장)

### 비밀번호 재설정

**경로**: `/forgot-password`

**절차**:

1. 등록된 이메일 주소 입력
2. 이메일로 전송된 재설정 링크 클릭
3. 새 비밀번호 입력 (`/reset-password/[token]`)
4. 재설정 완료 후 로그인

## 워크플로우 관리

**경로**: `/workflows`

### 개요

n8n에 등록된 워크플로우를 조회하고 실행할 수 있는 페이지입니다.

### 주요 기능

#### 1. 워크플로우 목록 조회

**화면 구성**:

```
┌─────────────────────────────────────────┐
│  Tags 목록                              │
│  ┌─────────────┐  ┌─────────────┐      │
│  │ Workflow 1  │  │ Workflow 2  │      │
│  │ 활성화됨    │  │ 비활성화됨  │      │
│  │ 5개 노드    │  │ 3개 노드    │      │
│  │ [실행]      │  │ [실행]      │      │
│  └─────────────┘  └─────────────┘      │
└─────────────────────────────────────────┘
```

**워크플로우 카드 정보**:

- 워크플로우 이름
- 활성화 상태 (활성화됨/비활성화됨)
- 노드 개수
- 태그 목록
- 생성일/수정일
- 실행 버튼
- "n8n에서 열기" 버튼

#### 2. 태그별 필터링

**방법 1: 사이드바에서 선택**

1. 좌측 메뉴에서 "Tags" 확장
2. 원하는 태그 클릭
3. 해당 태그의 워크플로우만 표시

**방법 2: 워크플로우 카드에서 클릭**

1. 워크플로우 카드의 태그 배지 클릭
2. 해당 태그로 자동 필터링

**필터 해제**:

- 사이드바에서 "모든 워크플로우" 클릭
- 또는 URL에서 `?tag=xxx` 파라미터 제거

#### 3. 워크플로우 실행

**절차**:

1. 활성화된 워크플로우 카드에서 "실행" 버튼 클릭
2. 워크플로우 실행 모달 표시
3. (선택) 입력 파라미터 설정
4. "실행" 버튼 클릭
5. 실행 결과 확인

**제약사항**:

- ⚠️ 비활성화된 워크플로우는 실행 불가
- ⚠️ n8n에서 먼저 활성화 필요

**실행 모달 기능**:

```
┌──────────────────────────────────────┐
│  워크플로우 실행                     │
│  ─────────────────────────────       │
│  워크플로우 이름: [Example WF]       │
│                                      │
│  입력 파라미터:                      │
│  ┌────────────────────────────────┐ │
│  │ Key: [입력]                    │ │
│  │ Value: [입력]                  │ │
│  └────────────────────────────────┘ │
│                                      │
│  [ 취소 ]  [ 실행 ]                 │
└──────────────────────────────────────┘
```

#### 4. n8n에서 열기

**기능**:

- 워크플로우 카드의 "n8n에서 열기" 버튼 클릭
- n8n UI의 워크플로우 목록 페이지가 새 탭에서 열림
- n8n에서 직접 워크플로우 편집 가능

#### 5. 지식 습득 프로세스

**경로**: 헤더의 "지식 습득 프로세스" 버튼

**기능**:

- 특정 워크플로우를 통해 지식을 학습하고 저장
- 모달 형태로 실행 인터페이스 제공
- 학습 결과를 MongoDB에 저장

### 사용 시나리오

**시나리오 1: 특정 태그의 워크플로우 실행**

1. 좌측 메뉴에서 "작업중" 태그 선택
2. 필터링된 워크플로우 목록 확인
3. 원하는 워크플로우의 "실행" 버튼 클릭
4. 실행 모달에서 파라미터 입력
5. "실행" 클릭 → 실행 시작

**시나리오 2: 워크플로우 편집 후 테스트**

1. "n8n에서 열기" 버튼 클릭
2. n8n UI에서 워크플로우 편집
3. 저장 후 gonsai2로 돌아옴
4. "새로고침" 버튼 클릭 (변경사항 반영)
5. "실행" 버튼으로 테스트

### 사용된 API

```
GET  /api/workflows           - 모든 워크플로우 조회 (태그 포함)
GET  /api/tags                - 모든 태그 조회
POST /api/workflows/:id/execute - 워크플로우 실행
```

### 팁 💡

- 좌측 메뉴의 Tags를 확장하면 모든 태그 목록을 볼 수 있습니다
- 워크플로우 카드의 태그 배지를 클릭하면 해당 태그로 즉시 필터링됩니다
- 비활성화된 워크플로우는 n8n에서 활성화해야 실행할 수 있습니다
- 새로고침 버튼을 클릭하여 최신 워크플로우 목록을 가져올 수 있습니다
- n8n에서 워크플로우에 태그를 추가/제거한 후 새로고침하면 변경사항이 반영됩니다

## 실행 내역 조회

**경로**: `/executions`

### 개요

워크플로우의 실행 기록을 조회하고 상세 정보를 확인할 수 있는 페이지입니다.

### 주요 기능

#### 1. 워크플로우 선택

**절차**:

1. 상단 필터에서 워크플로우 드롭다운 클릭
2. 조회할 워크플로우 선택
3. 해당 워크플로우의 실행 내역 자동 로드

**초기 상태**:

- 첫 번째 워크플로우의 실행 내역이 자동으로 로드됨
- "워크플로우 선택" 안내 메시지 표시 (워크플로우가 없는 경우)

#### 2. 필터 기능

**워크플로우 필터**:

- 드롭다운에서 워크플로우 선택
- 선택한 워크플로우의 실행만 표시

**상태 필터**:

- **모든 상태**: 전체 실행 내역
- **성공**: 정상 완료된 실행
- **실패**: 오류로 중단된 실행
- **실행 중**: 현재 진행 중인 실행
- **대기**: 실행 대기 중

**표시 개수 선택**:

- 10개, 20개, 50개, 100개 단위 선택 가능
- 기본값: 20개

#### 3. 실행 내역 카드

**표시 정보**:

```
┌─────────────────────────────────────────────┐
│  [아이콘] Workflow Name          [상태]    │
│  실행 ID: xxx-xxx-xxx                       │
│  ──────────────────────────────────────     │
│  시작 시간: 2025-01-12 14:30:45             │
│  종료 시간: 2025-01-12 14:31:02             │
│  실행 시간: 17초                            │
│  ──────────────────────────────────────     │
│  [ 상세 정보 ]  [ n8n에서 열기 ]           │
└─────────────────────────────────────────────┘
```

**상태 표시**:

- 🟢 **성공**: 녹색 배지 + 체크 아이콘
- 🔴 **실패**: 빨간색 배지 + X 아이콘
- 🔵 **실행 중**: 파란색 배지 + 회전 아이콘
- 🟡 **대기**: 노란색 배지 + 시계 아이콘

#### 4. 상세 정보 보기

**절차**:

1. 실행 카드에서 "상세 정보" 버튼 클릭
2. 카드 하단에 실행 데이터 표시 (JSON 형식)
3. 다시 "숨기기" 버튼 클릭하여 닫기

**실행 데이터 내용**:

- 워크플로우 입력 데이터
- 각 노드의 실행 결과
- 출력 데이터
- 오류 정보 (실패한 경우)

**JSON 뷰어 기능**:

- 구문 강조 표시
- 들여쓰기로 계층 구조 표현
- 최대 높이 제한 (스크롤 가능)

#### 5. n8n에서 열기

**기능**:

- "n8n에서 열기" 버튼 클릭
- n8n UI의 해당 실행 상세 페이지가 새 탭에서 열림
- n8n에서 실행 로그, 노드별 데이터 확인 가능

#### 6. 실행 시간 계산

**자동 계산 로직**:

```
실행 시간 = 종료 시간 - 시작 시간

표시 형식:
- 1시간 이상: "X시간 Y분 Z초"
- 1분 이상: "Y분 Z초"
- 1분 미만: "Z초"
```

**예시**:

- `2시간 15분 30초`
- `45분 10초`
- `15초`

### 사용 시나리오

**시나리오 1: 실패한 실행 찾기**

1. 워크플로우 선택
2. 상태 필터를 "실패"로 변경
3. 실패한 실행 목록 확인
4. 상세 정보 확장하여 오류 원인 분석
5. n8n에서 열어 상세 로그 확인

**시나리오 2: 성능 모니터링**

1. 워크플로우 선택
2. 실행 시간이 긴 실행 찾기
3. 상세 정보에서 느린 노드 확인
4. n8n에서 워크플로우 최적화

**시나리오 3: 최근 실행 확인**

1. 워크플로우 선택
2. 기본 정렬 (최신순) 확인
3. 새로고침으로 최신 상태 유지

### 사용된 API

```
GET /api/workflows                        - 워크플로우 목록 조회
GET /api/workflows/:id/executions?limit=N - 워크플로우 실행 내역 조회
```

### 팁 💡

- 워크플로우를 먼저 선택해야 실행 내역이 표시됩니다
- 상태 필터를 사용하여 실패한 실행만 빠르게 찾을 수 있습니다
- 실행 시간이 긴 워크플로우는 성능 최적화가 필요할 수 있습니다
- "n8n에서 열기" 버튼으로 n8n UI에서 실행 상세 내역을 확인할 수 있습니다
- 실행 데이터는 JSON 형태로 표시되며, 워크플로우의 입출력 데이터를 포함합니다

## 실시간 모니터링

**경로**: `/monitoring`

### 개요

n8n 워크플로우의 실시간 실행 상태, 로그, 메트릭을 모니터링하는 페이지입니다.

### 주요 기능

#### 1. WebSocket 연결 상태

**표시 위치**: 헤더 우측

**연결 상태**:

- 🟢 **연결됨**: 실시간 업데이트 가능
- 🟡 **연결 중...**: 서버 연결 시도 중
- 🔴 **연결 끊김**: 백엔드 서버 연결 불가

**자동 재연결**:

- 연결이 끊어지면 자동으로 재연결 시도
- 5초마다 연결 상태 체크

#### 2. 시스템 통계 대시보드

**4개의 통계 카드**:

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│ 총 실행     │ 성공률      │ 실패 횟수   │ 평균 실행   │
│ (24h)       │             │             │ 시간        │
│             │             │             │             │
│ 142         │ 95.8%       │ 6           │ 2.35s       │
│ 활성: 3개   │ 성공: 136건 │ 4.2%        │ 24h         │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

**통계 정보**:

- **총 실행 (24h)**: 최근 24시간 실행 횟수
- **성공률**: 성공한 실행 비율 (%)
- **실패 횟수**: 실패한 실행 개수 및 비율
- **평균 실행 시간**: 평균 소요 시간 (초 단위)

**자동 새로고침**:

- 1분마다 자동 갱신
- 페이지 로드 시 즉시 로드

#### 3. 실행 목록 (ExecutionList)

**위치**: 좌측 컬럼

**기능**:

- 실시간 실행 상태 표시
- WebSocket으로 실시간 업데이트
- 상태별 필터링 (실행 중, 성공, 실패)
- 각 실행의 시작 시간, 종료 시간, 실행 시간 표시

**실시간 업데이트 이벤트**:

- `execution.started`: 실행 시작 시 목록에 추가
- `execution.finished`: 실행 완료 시 상태 업데이트
- `execution.error`: 실행 실패 시 오류 표시

#### 4. 로그 스트림 (LogStream)

**위치**: 좌측 컬럼 하단

**기능**:

- 실시간 로그 메시지 표시
- WebSocket으로 실시간 업데이트
- 로그 레벨별 색상 구분 (info, warn, error)
- 자동 스크롤 (최신 로그 하단)

**로그 형식**:

```
[2025-01-12 14:30:45] [INFO] Workflow started: xxx
[2025-01-12 14:30:46] [WARN] Node slow response: yyy
[2025-01-12 14:30:50] [ERROR] Execution failed: zzz
```

**로그 색상**:

- 🔵 INFO: 파란색
- 🟡 WARN: 노란색
- 🔴 ERROR: 빨간색

#### 5. 메트릭 차트 (MetricsCharts)

**위치**: 우측 컬럼

**차트 종류**:

1. **실행 추이**: 시간대별 실행 횟수
2. **성공/실패율**: 성공 vs 실패 비율
3. **실행 시간 분포**: 실행 시간 히스토그램
4. **워크플로우별 실행 횟수**: 워크플로우 순위

**차트 기능**:

- 확대/축소
- 데이터 포인트 호버 시 상세 정보 표시
- 범례 클릭으로 데이터 시리즈 토글

#### 6. 알림 센터 (NotificationCenter)

**위치**: 헤더 우측

**기능**:

- 실시간 알림 표시
- 알림 개수 배지
- 알림 목록 드롭다운
- 알림 읽음 처리

**알림 종류**:

- 🔴 워크플로우 실행 실패
- 🟡 실행 시간 초과
- 🔵 워크플로우 활성화/비활성화
- 🟢 중요 실행 완료

### 사용 시나리오

**시나리오 1: 실시간 모니터링**

1. 모니터링 페이지 접속
2. WebSocket 연결 상태 확인 (🟢 연결됨)
3. 실행 목록에서 현재 실행 중인 워크플로우 확인
4. 로그 스트림에서 실시간 로그 확인
5. 메트릭 차트에서 전체 추이 파악

**시나리오 2: 문제 감지 및 해결**

1. 알림 센터에서 실패 알림 확인
2. 실행 목록에서 실패한 실행 찾기
3. 로그 스트림에서 오류 메시지 확인
4. `/executions` 페이지로 이동하여 상세 분석
5. n8n에서 워크플로우 수정

**시나리오 3: 성능 모니터링**

1. 시스템 통계 대시보드에서 평균 실행 시간 확인
2. 메트릭 차트에서 실행 시간 분포 확인
3. 느린 워크플로우 식별
4. 최적화 작업 수행

### 사용된 API

```
GET  /api/monitoring/stats                  - 시스템 통계 조회
WS   ws://backend-url                        - WebSocket 연결
     - execution.started                     - 실행 시작 이벤트
     - execution.finished                    - 실행 완료 이벤트
     - execution.error                       - 실행 오류 이벤트
     - log                                   - 로그 메시지 이벤트
```

### WebSocket 이벤트

**이벤트 목록**:

```javascript
// 실행 이벤트
execution.started     - 워크플로우 실행 시작
execution.finished    - 워크플로우 실행 완료
execution.error       - 워크플로우 실행 실패
execution.progress    - 실행 진행률 업데이트

// 워크플로우 이벤트
workflow.updated      - 워크플로우 업데이트
workflow.activated    - 워크플로우 활성화
workflow.deactivated  - 워크플로우 비활성화

// 로그 이벤트
log                   - 로그 메시지
```

## 문제 해결

### 일반적인 문제

#### 1. 로그인 실패

**증상**:

- "잘못된 인증 정보" 오류 메시지

**해결 방법**:

1. 이메일 주소 및 비밀번호 재확인
2. Caps Lock 확인
3. 비밀번호 재설정 시도
4. 브라우저 캐시 삭제 후 재시도

#### 2. 워크플로우 조회 실패

**증상**:

- "데이터 조회 중 오류 발생" 메시지
- 빈 워크플로우 목록

**해결 방법**:

1. 백엔드 서버 실행 상태 확인
   ```bash
   # 서버가 3000 포트에서 실행 중인지 확인
   curl http://192.168.0.50:3000/health
   ```
2. n8n API 키 확인 (백엔드 `.env` 파일)
3. n8n 서버 실행 상태 확인
4. 네트워크 연결 확인
5. 브라우저 콘솔에서 오류 메시지 확인

#### 3. 워크플로우 실행 실패

**증상**:

- "실행" 버튼 비활성화
- 실행 후 즉시 실패

**해결 방법**:

1. 워크플로우가 n8n에서 활성화되어 있는지 확인
2. n8n 서버 상태 확인
3. 워크플로우 설정 오류 확인 (n8n UI)
4. 입력 파라미터 유효성 확인
5. n8n 로그 확인

#### 4. WebSocket 연결 실패 (모니터링 페이지)

**증상**:

- "연결 끊김" 상태 표시
- 실시간 업데이트 안 됨

**해결 방법**:

1. 백엔드 서버 실행 확인
2. 환경 변수 확인
   ```bash
   # apps/frontend/.env.local
   NEXT_PUBLIC_WS_URL=ws://192.168.0.50:3000
   ```
3. 방화벽 설정 확인
4. 브라우저 콘솔에서 WebSocket 연결 오류 확인
5. 페이지 새로고침

#### 5. 실행 내역이 표시되지 않음

**증상**:

- "워크플로우를 선택하세요" 메시지
- 선택했지만 빈 목록

**해결 방법**:

1. 워크플로우를 드롭다운에서 선택했는지 확인
2. 해당 워크플로우의 실행 내역이 존재하는지 확인
3. 새로고침 버튼 클릭
4. 백엔드 API 연결 상태 확인

### 환경 변수 설정

**프론트엔드 환경 변수** (`apps/frontend/.env.local`):

```bash
# 백엔드 API URL
NEXT_PUBLIC_API_URL=http://192.168.0.50:3000

# WebSocket URL
NEXT_PUBLIC_WS_URL=ws://192.168.0.50:3000

# n8n UI URL
NEXT_PUBLIC_N8N_UI_URL=http://192.168.0.50:5678
```

**백엔드 환경 변수** (`apps/backend/.env`):

```bash
# MongoDB 연결
MONGODB_URI=mongodb://superadmin:password@localhost:27017/gonsai2?authSource=admin

# n8n API 설정
N8N_API_KEY=your-n8n-api-key
N8N_BASE_URL=http://localhost:5678

# 서버 포트
PORT=3000
HOST=0.0.0.0
```

### 브라우저 콘솔에서 디버깅

**절차**:

1. F12 키 또는 우클릭 > "검사" 클릭
2. "Console" 탭 선택
3. 오류 메시지 확인
4. 네트워크 요청 확인 ("Network" 탭)

**일반적인 오류**:

- `401 Unauthorized`: 인증 실패 (로그인 필요)
- `404 Not Found`: API 엔드포인트 없음 (URL 확인)
- `500 Internal Server Error`: 백엔드 서버 오류 (서버 로그 확인)
- `CORS Error`: CORS 설정 오류 (백엔드 CORS 설정 확인)
